🚀 Deployment Checklist — Fix "Failed to fetch"
1 Open your Apps Script project → Deploy → New deployment (or Manage deployments → Edit)
2 Set Execute as: Me and Who has access: Anyone — not "Anyone within domain" (GitHub Pages is external)
3 Copy the URL ending in /exec — not /dev (dev URLs block external access)
4 After any code change, you must create a new version — existing deployments do not auto-update
5 Grant permissions when prompted — click Review permissions → Advanced → Go to app (unsafe) → Allow
💡 Why GET not POST? Apps Script handles CORS automatically for GET requests. POST with Content-Type: application/json triggers a CORS preflight that Apps Script cannot respond to — causing "Failed to fetch". This portal uses GET with a ?payload= param to avoid this entirely.
